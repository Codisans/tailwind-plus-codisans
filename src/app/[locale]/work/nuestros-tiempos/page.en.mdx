## Data Collection: Multi-Platform, Resilient Scraping

The platform ingests data from 16 major Chilean news outlets â€” from Emol and CNN Chile to CIPER and El Mostrador.

Each outlet has a custom configuration layer, defining unique CSS selectors, exclusion rules, and crawl rates. The Puppeteer stealth stack was enhanced with user-agent rotation, request throttling, and adaptive retries to ensure reliable scraping without violating robots.txt directives.

Selective Processing Pipelines sanitize HTML, remove ads, normalize character encoding, and extract metadata. Each article is then queued for embedding generation and stored in the vector database.

## AI Engine: Semantic Understanding through Vector Embeddings

Traditional keyword search fails to detect conceptual relationships between articles â€” e.g., "housing reform" and "urban planning." To solve this, Codisans integrated Ollama's BGE-M3 Model, a multilingual embedding model capable of capturing semantic similarity in Spanish-language text.

- For every processed article, the system computes a dense 768-dimension embedding.
- These vectors are indexed in LibSQL's vector storage, allowing cosine similarity queries to find thematically related articles.

**Example:**

When one outlet reports "Tax Reform Negotiations" and another covers "Fiscal Pact Debate," the AI engine clusters them under one collection â€” even with zero shared keywords.

This unsupervised clustering enables Nuestros Tiempos to auto-generate "intelligent collections" that reflect how Chilean media frames complex issues differently.

## Performance & Orchestration

Scalability was critical. The backend was optimized for concurrent processing using Laravel Queues and Supervisor workers, ensuring multiple scrapers could run in parallel without blocking API responses.

Performance enhancements include:

- **Database Indexing:** Composite indexes for fast retrieval of article metadata.
- **Rate-Limiting Logic:** Enforced polite crawling intervals per domain.
- **Queue Isolation:** Separate queues for scraping, embedding, and clustering tasks.
- **Caching Layer:** In-memory caching for high-frequency vector queries.
- **Server-Side Rendering:** Implemented via Vite SSR for SEO and shareable previews.

The entire architecture was deployed with zero-downtime reloads via Laravel Forge and systemd integration.
